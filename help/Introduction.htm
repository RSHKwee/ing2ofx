Introduction


This (java) application converts (https://github.com/RSHKwee/ing2ofx/releases):

  1. ING (ING site) csv-file(s) (comma or semicolon separated).
    The csv-file(s) may contain transactions of multiple accounts.
  2. SNS (SNS Bank site) xml-files (CAMT.053 format).
    The xml-file(s) may contain transactions of multiple accounts.

To ofx files that can be read by a program like GnuCash (GnuCash).


The following steps are taken:

  1. The input files are read, the read transactions are stored in cache.
    1. The input files can be read all at once
    2. One by one
  2. The stored transactions are converted to OFX format (OFX.NET):
    1. A check is performed to avoid duplication of transactions.
    2. All stored transaction will be paired (if possible) with the following conditions:
      1. Same transaction date.
      2. Account To equals Account From or Account From equals Account To.
      3. Amounts of transactions sums to zero.
      The fit-id of the transactionsare modified, the paired transactions get the same fit-id.
  3. The OFX formatted transactions are stored in seperate files (one per account).
